Formule Cheie

Descompunere

Aditivă: $X_t = T_t + S_t + \varepsilon_t$

Multiplicativă: $X_t = T_t \times S_t \times \varepsilon_t$

Staționaritate

$\mathbb{E}[X_t] = \mu$ (constantă)

$\text{Var}(X_t) = \sigma^2$ (constantă)

$\text{Cov}(X_t, X_{t+h}) = \gamma(h)$

Model AR(p)

$X_t = c + \sum_{i=1}^{p}\phi_i X_{t-i} + \varepsilon_t$

Staționară dacă rădăcinile în afara cercului unitate

Model MA(q)

$X_t = \mu + \sum_{j=1}^{q}\theta_j\varepsilon_{t-j} + \varepsilon_t$

Întotdeauna staționară

ARIMA(p,d,q)

$\phi(L)(1-L)^d X_t = \theta(L)\varepsilon_t$

$d$ = ordinul diferențierii

SARIMA

$(p,d,q) \times (P,D,Q)_s$

ARIMA sezonier + non-sezonier

Model VAR(p)

$\mathbf{Y}_t = \mathbf{c} + \sum_{i=1}^{p}\mathbf{A}_i\mathbf{Y}_{t-i} + \boldsymbol{\varepsilon}_t$

Extensie multivariată a AR

Cauzalitate Granger

$X \rightarrow Y$: trecutul lui $X$ prezice $Y$

$H_0$: coeficienții lui $X$ = 0

GARCH(1,1)

$\sigma_t^2 = \omega + \alpha \varepsilon_{t-1}^2 + \beta \sigma_{t-1}^2$

Model pentru volatilitate

Value at Risk

$\text{VaR}_\alpha = \mu + z_\alpha \cdot \sigma_t$

Măsură de risc la nivel $\alpha$

Cointegrare

$Y_t = \beta X_t + u_t$, $u_t \sim I(0)$

Relație de echilibru pe termen lung

Selecția Modelului

AIC $= -2\log L + 2k$

BIC $= -2\log L + k\log n$

Care dintre următoarele NU este o caracteristică a datelor de tip serie de timp?

Întrebarea 5: Operatorul de Lag

Întrebarea 8: Proprietăți ale Prognozei

Pentru un model AR(1) staționar, ce se întâmplă cu prognozele pe măsură ce orizontul $h \to \infty$?

Corect! Acuratețea direcției = % din ori când sign($\Delta \hat{y}$) = sign($\Delta y$). Importantă pentru strategii de trading chiar dacă RMSE este mare.
Incorect. Răspunsul corect este B. Acuratețea direcției măsoară proporția de mișcări direcționale prezise corect.

Capitolul 9: Prophet și TBATS pentru Sezonalitate Multiplă

Întrebarea 1: Provocarea Sezonalității Multiple

De ce nu poate SARIMA standard să gestioneze datele orare de consum de electricitate?

  • A) SARIMA poate gestiona doar date lunare
  • B) SARIMA permite doar o singură perioadă sezonieră (parametrul $m$)
  • C) SARIMA nu suportă componente de trend
  • D) SARIMA necesită date distribuite normal
Corect! Datele orare au pattern-uri zilnice (24h), săptămânale (168h) și anuale (8760h). Parametrul unic $m$ al SARIMA nu poate capta toate acestea simultan.
Incorect. Răspunsul corect este B. SARIMA permite doar o singură perioadă sezonieră, făcându-l nepotrivit pentru sezonalitate multiplă.

Întrebarea 2: Acronimul TBATS

Ce înseamnă TBATS?

  • A) Trend, Baseline, ARMA, Transform, Seasonal
  • B) Trigonometric, Box-Cox, ARMA, Trend, Seasonal
  • C) Time-Based Automatic Time Series
  • D) Temporal Bayesian Adaptive Trend System
Corect! TBATS: Trigonometric (termeni Fourier), Box-Cox (stabilizarea varianței), ARMA (erori), Trend (local amortizat), Seasonal (perioade multiple).
Incorect. Răspunsul corect este B. Fiecare literă reprezintă o componentă a modelului.

Întrebarea 3: Termeni Fourier

În TBATS, creșterea numărului de armonici Fourier ($K$) pentru un pattern sezonier:

  • A) Îmbunătățește întotdeauna acuratețea prognozei
  • B) Permite forme sezoniere mai flexibile (complexe)
  • C) Reduce complexitatea modelului
  • D) Elimină necesitatea transformării Box-Cox
Corect! Mai multe armonici = mai multă flexibilitate dar și mai mulți parametri (risc de supraajustare). Maximum util $K = m/2$.
Incorect. Răspunsul corect este B. Mai mulți termeni Fourier permit pattern-uri sezoniere mai complexe.

Întrebarea 4: Descompunerea Prophet

Prophet descompune o serie de timp în care componente?

  • A) Componente AR, MA și sezoniere
  • B) Trend, sezonalitate, sărbători și eroare
  • C) Medie, varianță și autocorelație
  • D) Nivel, pantă și curbură
Corect! Prophet: $y(t) = g(t) + s(t) + h(t) + \varepsilon_t$ unde $g$ = trend, $s$ = sezonalitate, $h$ = sărbători.
Incorect. Răspunsul corect este B. Prophet folosește un model aditiv cu trend, sezonalitate, sărbători și eroare.

Întrebarea 5: Prophet vs TBATS

Când ai alege Prophet în loc de TBATS?

  • A) Când ai nevoie de selecție automată a modelului
  • B) Când ai sărbători cunoscute și puncte de schimbare de încorporat
  • C) Când ai nevoie de cel mai parsimonios model
  • D) Când datele tale nu au trend
Corect! Prophet excelează la încorporarea cunoștințelor din domeniu: sărbători, evenimente, puncte de schimbare cunoscute. TBATS este mai bun pentru modelare automată.
Incorect. Răspunsul corect este B. Puterea Prophet este integrarea ușoară a cunoștințelor din domeniu.

Întrebarea 6: Modul de Sezonalitate

Pentru vânzări retail unde vânzările din decembrie sunt de 3x media lunară, care mod de sezonalitate este mai potrivit în Prophet?

  • B) Multiplicativ - sezonalitatea scalează cu nivelul seriei
  • C) Nu contează - Prophet alege automat
  • D) Aditiv - este întotdeauna mai stabil
Corect! Când efectele sezoniere cresc cu nivelul (vânzările de Crăciun sunt 3x, nu +$1000), multiplicativ este mai potrivit.
Incorect. Răspunsul corect este B. Sezonalitatea multiplicativă este potrivită când variațiile sezoniere sunt proporționale cu nivelul seriei.

Întrebarea 7: Puncte de Schimbare

Incorect. Răspunsul corect este B. Changepoints în Prophet se referă la schimbări în rata de creștere a trendului.

Întrebarea 8: Box-Cox în TBATS

Care este scopul transformării Box-Cox în TBATS?

Incorect. Răspunsul corect este B. Box-Cox este folosit pentru stabilizarea varianței în TBATS.

Capitolul 5: Modele de Volatilitate ARCH/GARCH

Întrebarea 1: Volatility Clustering

Ce reprezintă „volatility clustering"?

  • C) Randamentele sunt corelate în timp
  • D) Distribuția randamentelor este normală
Corect! Volatility clustering = perioade agitate urmate de perioade agitate.
Incorect. Răspunsul corect este B. Volatilitatea tinde să persiste.

Întrebarea 2: Parametrii GARCH

În GARCH(1,1): σₜ² = ω + α·εₜ₋₁² + β·σₜ₋₁², ce reprezintă α?

  • A) Persistența volatilității
  • B) Nivelul de bază al volatilității
  • C) Reacția la șocuri recente (news coefficient)
  • D) Varianța necondiționată
Corect! α măsoară cât de mult reacționează volatilitatea la șocuri noi.
Incorect. Răspunsul corect este C. α = reacția la news.

Întrebarea 3: Condiția de Stationaritate

Care este condiția de stationaritate pentru GARCH(1,1)?

  • A) ω > 0
  • B) α + β = 1
  • C) α + β < 1
  • D) α > β
Corect! Pentru stationaritate, α + β < 1 (persistența totală sub 1).
Incorect. Răspunsul corect este C. Stationaritate: α + β < 1.

Întrebarea 4: Varianța Necondiționată

Care este formula varianței necondiționate în GARCH(1,1)?

  • A) σ̄² = ω
  • B) σ̄² = ω / (1 - α)
  • C) σ̄² = ω / (1 - α - β)
  • D) σ̄² = ω / (α + β)
Corect! Varianța necondiționată = ω / (1 - α - β).
Incorect. Răspunsul corect este C.

Întrebarea 5: Leverage Effect

Ce este „leverage effect"?

  • A) Șocurile pozitive cresc volatilitatea mai mult
  • B) Șocurile negative cresc volatilitatea mai mult decât cele pozitive
  • C) Volatilitatea este independentă de semnul șocurilor
  • D) Randamentele sunt asimetrice
Corect! Scăderile de preț măresc volatilitatea mai mult decât creșterile.
Incorect. Răspunsul corect este B. Leverage = asimetrie negativă.

Întrebarea 6: EGARCH

În EGARCH, un parametru γ negativ indică:

  • A) Absența leverage effect
  • B) Prezența leverage effect
  • C) Volatilitate constantă
  • D) Model nestationar
Corect! γ < 0 în EGARCH confirmă leverage effect.
Incorect. Răspunsul corect este B.

Întrebarea 7: Avantajul EGARCH

Care este principalul avantaj al EGARCH față de GARCH?

  • A) Este mai rapid de estimat
  • B) Nu necesită restricții de non-negativitate
  • C) Are mai puțini parametri
  • D) Este mai ușor de interpretat
Corect! EGARCH modelează ln(σ²), deci σ² > 0 automat.
Incorect. Răspunsul corect este B.

Întrebarea 8: Testul ARCH-LM

Ce test folosim pentru a detecta efecte ARCH în reziduuri?
